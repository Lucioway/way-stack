# create-agent — SDD Agent Factory

Meta-skill that turns a workflow description into a working agent via Spec-Driven Development (GSD + Ralph Loop + BMAD fused).

## When to use

User says any of:
- "build me an agent that…"
- "I want to automate X"
- "create a bot that does Y"
- "make a workflow for Z"
- Describes a repetitive manual task

## Phase 0 — Project Intake

Ask AT MOST 5 targeted questions. Skip any that the user already answered unprompted.

1. **What to automate?** — the specific workflow in one sentence
2. **Platforms involved?** — email / WhatsApp / web / API / database / files
3. **Output?** — report, message, file, API call, DB write
4. **Frequency?** — real-time / scheduled / on-demand
5. **Users?** — just you / team / external customers

### Stack hints by platform

| Platform | Recommended approach |
|---|---|
| Email (read/send) | `imaplib` (built-in) or Gmail API |
| WhatsApp | Selenium + WhatsApp Web (persistent session) |
| Reddit, TikTok, YouTube, HN, GitHub, Instagram, Twitter/X, Yahoo Finance | OpenCLI (`opencli list` to verify) — zero-token structured output |
| Custom web (Amazon, Meta Ads, Trustpilot, etc.) | Selenium + Brave/Chrome with CSS selectors centralized |
| Text analysis | Claude API (Sonnet for simple, Opus for complex reasoning) |
| Storage | Notion API, Google Drive, local files, SQLite |

## Scaffold standard files

Every agent generated by this skill MUST produce:

- `CLAUDE.md` — project rules (this is read first every session)
- `.env.example` — env var template
- `.gitignore` — excludes secrets, caches, venv
- `requirements.txt` (Python) or `package.json` (Node)
- `SETUP_GUIDE.md` — step-by-step for the user

## Hard rules

1. Never implement without a spec — if no `SPEC.md`, run `/agent-spec` first
2. Never write more than 3 tasks per batch
3. Never skip verification
4. Never accumulate context — each task runs fresh
5. Always commit atomically — one task = one commit
6. Always dry-run before live (`--dry-run`, `--once` flags)
7. SPEC.md is the source of truth — code is derivative

## Handoff

After `PROJECT_BRIEF.md` is written + committed:

```
✓ Phase 0 complete. Next: /agent-spec
```

Do NOT continue into Phase 1 inside the same skill invocation. The SDD flow requires context reset between phases.

## Design principles (from the source framework)

1. Plan Mode before Build — analyze before code
2. One minute of planning saves 10 minutes of build
3. Skills > MCP for efficiency — skills consume fewer tokens
4. Module by module — build and test one piece at a time
5. Dry-run always before live
6. Context management — concise prompts, high info density
7. Robust error handling — retry with backoff, local backup, logs
8. Centralized CSS selectors for scrapers
9. Single `config.py` / `config.ts`
10. Rate limiting — respect every service's limits
11. OpenCLI before Selenium when supported
12. Spec-driven — docs are source of truth, code is derived
13. Fresh context per task — never accumulate conversation
